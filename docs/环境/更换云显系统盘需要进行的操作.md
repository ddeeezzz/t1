# 更换云显系统盘需要进行的操作 (Environment Recovery Guide)

当云平台更换实例型号（如 A16 切 4090D）或重置系统盘后，请按照以下步骤快速恢复项目运行环境。所有代码和模型数据均已保存在挂载盘 `~/data` 中，此处仅需恢复系统级配置。

## 1. 安装必备系统工具
进入新机器后，首先安装下载神器和后台神器：
```bash
apt-get update && apt-get install tmux aria2 ffmpeg -y
```

说明：
- 本环境可能没有 `sudo` 命令，直接使用 `apt-get ...` 即可（前提是当前账号有对应权限）。
- `ffmpeg`/`ffprobe` 是音频解码链路的系统依赖，模块 A 解析 `m4a` 时必须可用。

安装后立刻验证：
```bash
ffmpeg -version
ffprobe -version
```

进一步验证项目音频可被解码（以 `wuli.m4a` 为例）：
```bash
ffprobe -v error -show_entries format=filename,format_name,duration -show_streams ~/data/t1/resources/wuli.m4a
```

预期至少包含：
- `codec_name=aac`（音频流存在）
- `format_name=mov,mp4,m4a,3gp,3g2,mj2`

## 1.1 `bypy` 导入要做的事（简要：安装 `rg`）
如果不走 `curl` 下载，可以直接用项目虚拟环境里的 `bypy` 从网盘拉取 `ripgrep` 二进制包并安装。

先确认使用的是 `t1` 的 `bypy`：
```bash
/root/data/t1/.venv/bin/bypy --version
```

可先看一下远端是否已有安装包（示例放在 `环境` 目录）：
```bash
/root/data/t1/.venv/bin/bypy list 环境
```

下载并安装 `rg`（示例文件：`ripgrep-14.1.0-x86_64-unknown-linux-musl.tar.gz`）：
```bash
cd ~/data
/root/data/t1/.venv/bin/bypy downfile 环境/ripgrep-14.1.0-x86_64-unknown-linux-musl.tar.gz ./ripgrep-14.1.0-x86_64-unknown-linux-musl.tar.gz
tar xf ripgrep-14.1.0-x86_64-unknown-linux-musl.tar.gz
cp ripgrep-14.1.0-x86_64-unknown-linux-musl/rg /usr/bin/
rm -rf ripgrep-14.1.0-x86_64-unknown-linux-musl ripgrep-14.1.0-x86_64-unknown-linux-musl.tar.gz
rg --version
```

说明：
- 这里显式使用 `/root/data/t1/.venv/bin/bypy`，避免误用系统环境里的 `bypy`。
- 如果你的远端目录或文件名不同，把 `downfile` 后面的路径替换成实际路径即可。

## 2. 配置 `~/.bashrc`（核心：全局自动激活 `t1/.venv`）
使用 `vim ~/.bashrc`（或直接通过编辑器）打开配置文件，在文件**最末尾**追加以下内容。

这段配置的重点是：打开交互式 shell 后，直接激活 `~/data/t1/.venv`，不再按目录判断，避免误用系统 Python 或 conda base 环境。

```bash
# ==========================================
#[T1 项目专属环境配置]
# ==========================================

# 1. 环境变量与防爆盘机制 (极其重要)
# 开启 HuggingFace 国内镜像加速
export HF_ENDPOINT="https://hf-mirror.com"
# 强制把 HF 下载缓存转移到数据盘，防止系统盘撑爆导致死机
export HF_HOME="$HOME/data/hf_cache"
# 强制把 uv 的包缓存转移到数据盘
export UV_CACHE_DIR="$HOME/data/uv_cache"

# 2. 快捷指令别名
alias cdt1="cd $HOME/data/t1"
alias watchgpu="watch -n 1 nvidia-smi"
alias logs="tail -f $HOME/data/t1/log/*.log"

# 3. 全局自动激活 t1 虚拟环境：进入任意目录都生效
if [[ $- == *i* ]]; then
    t1_venv="$HOME/data/t1/.venv"
    if [[ -f "$t1_venv/bin/activate" ]] && [[ "${VIRTUAL_ENV:-}" != "$t1_venv" ]]; then
        # 解决与默认 Conda 环境的冲突
        if type conda >/dev/null 2>&1; then
            conda deactivate >/dev/null 2>&1 || true
        fi
        # shellcheck disable=SC1091
        source "$t1_venv/bin/activate"
    fi
fi
# ==========================================
```

## 3. 使配置生效并验证
修改完 `~/.bashrc` 后，执行以下命令立即生效：
```bash
source ~/.bashrc
```

在任意目录执行，确认当前 Python 已是 `t1/.venv`：
```bash
cd ~
echo "$VIRTUAL_ENV"
which python
python -V
```

预期结果：
- `VIRTUAL_ENV` 指向 `~/data/t1/.venv`
- `which python` 指向 `~/data/t1/.venv/bin/python`

## 4. 恢复 Python 依赖 (如果需要)
新系统盘可能没有 `uv` 或缺少部分系统级依赖。进入项目目录后，如果运行报错，执行一次环境同步：
```bash
cdt1
pip install uv  # 如果系统没自带的话
uv sync         # 瞬间按 uv.lock 对齐所有依赖库
```

环境恢复完成！

## 5. 本地查看任务监督面板（SSH 端口转发）
在云显服务器上启动 `mvpl monitor` 后，监督服务通常监听在服务器本机 `127.0.0.1:45705`。  
如需在你自己的电脑浏览器里访问，先在**本地电脑终端**执行：

```bash
ssh -N -L 45705:127.0.0.1:45705 -p 31903 root@ssh-cn-huabei1.ebcloud.com
```

说明：
- `-L 45705:127.0.0.1:45705`：把你本地 `45705` 端口转发到远端服务器的 `127.0.0.1:45705`。
- `-N`：只做端口转发，不执行远端命令。
- 保持该终端窗口不要关闭；关闭即断开转发。
- 转发建立后，在本地浏览器打开：`http://127.0.0.1:45705/task-monitor?task_id=<你的任务ID>`。
