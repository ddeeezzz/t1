### A+D 联合改造计划：防漂移帧锁定 + 器乐突变卡点

#### Summary
在不改模块契约与配置结构的前提下，同时落地两项升级：
1. 模块D：从“按秒截断”改为“按全局绝对帧分配”，消除多片段拼接的累积漂移。  
2. 模块A：器乐段卡点从“RMS峰值优先”升级为“瞬态突变优先 + 峰值回退”，更贴合 Kick/Drop 爆发点。  

验收标准按你确认的口径：`24fps` 下总时长误差控制在 `<= 1 帧`。

#### Key Changes
- **模块D时间轴防漂移（`src/music_video_pipeline/modules/module_d.py`）**
  - 在 `_render_segment_videos` 中不再使用片段 `-t`，改为 `-frames:v`。
  - 新增“全局帧分配”逻辑（建议新增内部函数，如 `_allocate_segment_frames_by_timeline`）：
    - 输入：`frame_items` 的 `start_time/end_time`、`audio_duration`、`fps`。
    - 规则：
      1. 目标总帧 `target_total = round(audio_duration * fps)`。
      2. 用每段绝对边界（`end_time`）映射到帧边界，并做单调修正，得到每段帧数差分。
      3. 帧数总和必须等于 `target_total`（不允许累积偏差）。
    - 产出：每段 `exact_frames`，用于 FFmpeg `-frames:v`。
  - 保留 concat 与最终 `-t audio_duration` 裁切，作为最后兜底，不改变现有输出链路。
  - 日志策略按你选择：默认打印汇总（目标总帧、分配总帧、偏差），仅在异常/失败时打印逐段明细。

- **模块A器乐卡点升级（`src/music_video_pipeline/modules/module_a.py`）**
  - 新增 `_rms_delta_at(time_value, rms_times, rms_values, window_ms=100.0)`：
    - 计算 `max(0, rms(t) - rms(t-0.1s))`，衡量正向爆发强度。
  - 在 `_select_small_timestamps` 的 `instrumental_set` 分支中替换主逻辑：
    - `onset_in_segment` 存在时，优先取 `argmax(_rms_delta_at)`。
    - 若该段所有 delta 近似 0（无突变）则回退到原峰值策略（`_rms_value_at`）或节拍中点。
  - 器乐段附加节拍密度调整为你提供的方向：`len>4` 时用 `[::2]`，否则保留全部，减少漏关键点风险。

- **接口与兼容**
  - `ModuleAOutput`、`ModuleBOutput`、`module_c_output.json` 字段不变。
  - 不新增配置项，不改 CLI 参数，不改状态机语义。

#### Test Plan
- **新增/强化单元测试（建议放在 `tests/test_module_a_algorithm.py` 与新增 `tests/test_module_d_timing.py`）**
  - 模块D：
    1. 给定多段 `start/end`，验证分配帧数总和严格等于 `round(audio_duration*fps)`。
    2. 验证每段命令包含 `-frames:v` 且不再依赖片段级 `-t`。
    3. 验证边界单调与无负帧数（异常输入需抛清晰错误）。
  - 模块A：
    1. 构造“峰值最大点 ≠ 突变最大点”的 RMS/onset 样本，断言选择突变最大点。
    2. 构造“无明显突变”样本，断言触发峰值/节拍回退。
    3. 验证器乐段附加节拍抽样策略已从 `[::4]` 改为动态规则。
- **回归**
  - 运行现有全量测试。
  - 跑一次真实链路 `mvpl run`，核对：
    - 任务 `A/B/C/D` 全 `done`；
    - 最终视频时长与音频时长偏差 `<= 1帧`（24fps 即 41.67ms）。

#### Assumptions
- `frame_items` 由上游保证按时间顺序且 `start/end` 基本连续（当前链路满足）。
- 输出仍以 24fps 为主，不通过提高 fps 达成 10ms 级误差目标。
- 本轮仅做算法与调度精度修复，不引入配置化开关与新外部依赖。
