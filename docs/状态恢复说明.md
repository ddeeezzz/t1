# 状态恢复说明（resume）

## 1. 状态机定义

每个模块（A/B/C/D）状态只允许以下取值：

- `pending`：待执行
- `running`：执行中
- `done`：执行完成
- `failed`：执行失败

任务总状态同样使用这四个状态。

## 2. 恢复语义

`resume` 的恢复逻辑是：

1. 查询模块顺序 `A -> B -> C -> D`。
2. 找到第一个不是 `done` 的模块。
3. 从该模块继续执行到 `D`。
4. 已经 `done` 的上游模块不会重跑。

## 3. 使用方法

```powershell
uv run mvpl resume --task-id demo_20s --config configs/default.json
```

## 4. 强制从指定模块恢复

当你希望忽略历史结果，从某个模块重新开始：

```powershell
uv run mvpl resume --task-id demo_20s --config configs/default.json --force-module B
```

说明：`--force-module B` 会将 `B/C/D` 重置为 `pending`，`A` 保持原状态。

## 5. 如何定位失败原因

1. 查看命令行日志，关注 `模块X执行失败` 行。
2. 查看状态库 `runs/pipeline_state.sqlite3` 中的 `module_runs.error_message`。
3. 查看该任务目录下中间产物是否完整：
- `runs/<task_id>/artifacts/module_a_output.json`
- `runs/<task_id>/artifacts/module_b_output.json`
- `runs/<task_id>/artifacts/module_c_output.json`

## 6. 建议排障顺序

1. 先确认输入音频路径正确。
2. 再确认 FFmpeg 可执行。
3. 再检查失败模块对应的中间文件是否存在且格式正确。
4. 最后使用 `run-module` 单独调试故障模块。


> 正式全名命令：uv run music-video-pipeline ...（与 uv run mvpl ... 等价）。
