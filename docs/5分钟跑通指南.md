# 5分钟跑通指南（MVP）

## 1. 环境准备

1. 安装 Python 3.11+。
2. 安装 `uv`（用于依赖管理与执行）。
3. 确保本机已安装 `ffmpeg`（并可在命令行直接执行 `ffmpeg`）。

## 2. 安装依赖

在 `t1/` 目录执行：

```powershell
uv sync
```

> 依赖说明：`funasr` 已按官方 README 对齐为 Git 固定版本（锁定 `modelscope/FunASR` 指定 commit），用于确保 `Fun-ASR-Nano-2512` 的 `FunASRNano` 注册可用，避免 PyPI 同版本内容差异导致的运行时不一致。

如果你要运行测试：

```powershell
uv sync --extra test
```

## 3. 执行全链路（20秒样片）

默认输入是 `resources/juebieshu20s.mp3`，执行命令：

```powershell
uv run mvpl run --task-id demo_20s --config configs/default.json
```

> 配置升级提醒：`module_a.funasr_language` 为必填项，需在配置文件中显式设置（如 `"auto"`、`"zh"`、`"en"`、`"ja"`）。  
> 日语样例 `configs/jieranduhuo.json` 建议固定为 `"ja"`，可显著降低自动语言检测误判。

输出位置：

- 任务目录：`runs/demo_20s/`
- 中间产物：`runs/demo_20s/artifacts/`
- 最终视频：`runs/demo_20s/final_output.mp4`
- 状态库：`runs/pipeline_state.sqlite3`

## 4. 常用调试命令

执行单模块：

```powershell
uv run mvpl run-module --task-id demo_20s --module A --config configs/default.json
```

从指定模块强制重跑（该模块及下游会重置）：

```powershell
uv run mvpl run --task-id demo_20s --config configs/default.json --force-module C
```

## 5. 常见报错与处理

1. `找不到 ffmpeg 可执行文件`  
原因：系统未安装 FFmpeg 或 PATH 未配置。  
处理：安装 FFmpeg，或在 `configs/default.json` 中把 `ffmpeg_bin` 改为绝对路径。

2. `音频文件不存在`  
原因：传入的 `--audio-path` 错误。  
处理：检查路径是否相对 `t1/` 或直接使用绝对路径。

3. `上游模块未完成`  
原因：直接运行下游模块但上游不是 `done`。  
处理：先执行上游模块，或使用 `run` 全链路运行。

4. `ModuleAConfig.__init__() missing 1 required positional argument: 'funasr_language'`  
原因：配置文件缺少 `module_a.funasr_language`。  
处理：在配置中补充该字段，例如 `"funasr_language": "auto"`。

5. `FunASR 模型注册失败（FunASRNano 未注册）`  
原因：`funasr` 运行时版本与 Nano 注册表不匹配（常见于未按仓库锁定版本安装）。  
处理：执行 `uv sync` 以安装仓库锁定的 Git 依赖；随后重跑模块 A，日志中应看到 FunASR 正常初始化并产出 `lyric_units`。

> 正式全名命令：uv run music-video-pipeline ...（与 uv run mvpl ... 等价）。
