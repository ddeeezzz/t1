# 5分钟跑通指南（MVP）

## 1. 环境准备

1. 安装 Python 3.11+。
2. 安装 `uv`（用于依赖管理与执行）。
3. 确保本机已安装 `ffmpeg`（并可在命令行直接执行 `ffmpeg`）。

## 2. 安装依赖

在 `t1/` 目录执行：

```powershell
uv sync
```

如果你要运行测试：

```powershell
uv sync --extra test
```

## 3. 执行全链路（20秒样片）

默认输入是 `resources/juebieshu20s.mp3`，执行命令：

```powershell
uv run mvp-pipeline run --task-id demo_20s --config configs/default.json
```

输出位置：

- 任务目录：`runs/demo_20s/`
- 中间产物：`runs/demo_20s/artifacts/`
- 最终视频：`runs/demo_20s/final_output.mp4`
- 状态库：`runs/pipeline_state.sqlite3`

## 4. 常用调试命令

执行单模块：

```powershell
uv run mvp-pipeline run-module --task-id demo_20s --module A --config configs/default.json
```

从指定模块强制重跑（该模块及下游会重置）：

```powershell
uv run mvp-pipeline run --task-id demo_20s --config configs/default.json --force-module C
```

## 5. 常见报错与处理

1. `找不到 ffmpeg 可执行文件`  
原因：系统未安装 FFmpeg 或 PATH 未配置。  
处理：安装 FFmpeg，或在 `configs/default.json` 中把 `ffmpeg_bin` 改为绝对路径。

2. `音频文件不存在`  
原因：传入的 `--audio-path` 错误。  
处理：检查路径是否相对 `t1/` 或直接使用绝对路径。

3. `上游模块未完成`  
原因：直接运行下游模块但上游不是 `done`。  
处理：先执行上游模块，或使用 `run` 全链路运行。
