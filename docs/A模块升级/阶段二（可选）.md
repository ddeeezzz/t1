## 阶段二（可选）

### 1. 时间戳增强方向

未来可以增加“LRC 时间戳与模块 A beat array 对齐/吸附”的增强能力，用于处理部分歌曲同步歌词存在系统性提前量、精度不稳定或与内部节拍时间轴不一致的问题。

增强方向可包括：

1. 获取 LRCLIB 返回的底层 LRC 时间戳。
2. 将这些时间点与模块 A 原有的 madmom/allin1 输出的 beat array 做最近邻对齐。
3. 执行 snap to nearest beat。
4. 输出吸附后的时间点作为增强版歌词时间戳结果。

说明：

- 该能力当前仅作为后续增强预留，不属于本次必做范围；
- 后续若实现，建议设计为可配置策略，而不是默认强制开启；
- 在确认大多数歌曲确有必要时，再决定是否将其升级为默认流程。

### 2. 纯文本歌词的后续扩展

以下方案保留为后续可扩展方向，当前阶段不进入实现范围：

- 当 LRCLIB 只有纯文本歌词时，使用 LLM 对 FunASR 输出的歌词文本做矫正；
- 保留 FunASR 时间戳，仅替换文本内容；
- 基于纯文本歌词做重型歌词对齐或额外文本对齐增强。
